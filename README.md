
# AI Research Navigator

## Overview

AI Research Navigator is a citation-grounded Retrieval-Augmented Generation (RAG) system designed to help users explore AI and Machine Learning literature through natural language queries.

The project was built to address a common problem with modern AI assistants: they often provide answers without clear evidence, fabricate references, or confidently generate incorrect information. AI Research Navigator takes a different approach by grounding every response in a curated research corpus and attaching citations to the supporting sources.

The system focuses on transparency, traceability, and reliability rather than open-domain question answering.

---

## Motivation

Students and researchers frequently rely on AI tools to understand research papers, surveys, and technical concepts. While these tools are convenient, they often struggle to answer an important question:

**"Where did this information come from?"**

AI Research Navigator was developed to ensure that answers are always supported by retrieved documents from a known corpus. If sufficient evidence cannot be found, the system refuses to answer rather than generating unsupported information.

---

## Research Corpus

The corpus consists of AI and Machine Learning resources including:

* Research papers
* Hugging Face NLP course material
* Technical blogs
* Survey articles
* Educational resources

Each document is parsed, segmented into chunks, enriched with metadata, embedded, and stored in a vector database for retrieval.

---

## Retrieval Approach

The retrieval system combines semantic and lexical search techniques.

### Dense Retrieval

Dense retrieval uses sentence embeddings generated by Sentence Transformers. This allows the system to retrieve conceptually relevant information even when the wording of a query differs from the wording of the source document.

### BM25 Retrieval

BM25 retrieval complements dense retrieval by performing keyword-based matching. This improves performance for technical terminology, paper titles, and exact phrase searches.

### Hybrid Fusion

The final ranking combines both approaches:

* Dense retrieval contributes semantic relevance.
* BM25 contributes lexical relevance.

This hybrid strategy consistently performs better than either approach alone.

---

## Citation Grounding

A core objective of the project is ensuring that generated answers remain traceable to source material.

Every answer is generated using retrieved evidence and includes citations derived from the corresponding documents.

Additional citation handling includes:

* Citation generation
* Citation deduplication
* Source attribution
* Reference formatting

This allows users to verify information independently.

---

## Refusal Mechanism

Traditional language models often attempt to answer questions even when insufficient evidence exists.

AI Research Navigator instead evaluates retrieval confidence before answer generation.

When the system cannot locate enough supporting evidence within the corpus, it returns a refusal response rather than generating potentially misleading information.

This design choice prioritizes reliability over answer coverage.

---

## Agent-Based Query Routing

The project includes a LangGraph-based routing layer that directs queries to specialized workflows.

Implemented capabilities include:

* Concept explanation
* Paper deep dives
* Method comparison
* Recent developments discovery
* Paper recommendation
* Out-of-scope detection

This allows different query types to be handled using workflows tailored to the user's intent.

## Running the Project

Install dependencies:

```bash
uv sync
```

Run ingestion:

```bash
python main.py
```

Run tests:

```bash
uv run pytest src/tests
```

Run type checking:

```bash
uv run mypy src
```

Run evaluation:

```bash
python -m src.evaluation.run_evaluation
```
